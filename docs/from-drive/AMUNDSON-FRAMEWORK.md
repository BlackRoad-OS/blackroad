## Overview

The Amundson Framework proposes a novel ontological foundation where **four fundamental primitives** and their permutations generate physical domains and computational architectures:

| Primitive | Symbol | Meaning |
|---|---|---|
| **Structure** | 1 | Static form, substrate |
| **Change** | 2 | Dynamics, transformation |
| **Strength** | 3 | Force, binding, intensity |
| **Scale** | 4 | Hierarchy, emergence level |

## Comparison with Existing Unification Frameworks

### Wheeler's "It from Bit" (1989)
- Information as substrate of physical reality
- "Every item of the physical world has at bottom an immaterial source and explanation"
- Maps to 1-2-3-4: binary = Structure(1), measurement events = Change(2)
- **Critical gap:** lacks Strength(3) and Scale(4), no dynamical mechanism for bits → spacetime geometry

### Wolfram's Computational Universe / Ruliad (2020)
- Ruliad = "the entangled limit of everything computationally possible"
- Derives spacetime, quantum mechanics, gravity from hypergraph rewriting rules
- Maps to 1-2-3-4: hypergraph states = Structure, rule applications = Change, causal structure = implicit Strength, observer-dependent slicing = Scale
- **Key distinction from Amundson:** Wolfram explores ALL possible rules (infinite), Amundson identifies MINIMAL sufficient ontological primitives
- Achieves background independence but lacks specificity — no unique rule for our universe identified

### String Theory / M-Theory
- Extra dimensions compactified to generate apparent 4D
- Naturally provides Strength(3) via coupling constants and Scale(4) via compactification
- **Critical gap:** no non-perturbative definition, exponential landscape problem (10^500 vacua)

### Loop Quantum Gravity (LQG)
- Spacetime quantized at Planck scale via spin networks
- Structure(1) = spin network nodes, Change(2) = spin foam evolution, Strength(3) = coupling at vertices
- **Critical gap:** no derivation of Standard Model particles, Scale(4) emergence unclear

### Integrated Information Theory (IIT)
- Consciousness = integrated information (Φ) above threshold
- Maps: Structure(1) = system architecture, Strength(3) = Φ value
- **Critical gap:** computational intractability for large systems
